# 웹학습지 1 개발 지침
## 나를 알아봐요 — 도입 & 자기탐구

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| 파일명 | `worksheet1.html` |
| 대상 | 3학년 도덕 1단원 1차시 (나를 알아봐요) |
| 사용자 | 학생 개인 |
| 사용 시간 | 도입 5분 + 전개1 10분 = 약 15분 |
| 사용 기기 | 크롬북 (카메라 내장), 태블릿, PC |
| 저장 방식 | localStorage 자동 저장 + PDF/인쇄 내보내기 |

## 2. 수업 흐름 대응

```
[도입 5분]  거울 보기 (크롬북 카메라 실행)
    ↓
[도입 연결] 나를 나타내는 낱말 3개 태그 입력
    ↓
[전개1 10분] 나는 이런 사람이에요 — 5개 문장 완성
    ↓
[마무리]     오늘 내가 발견한 나의 모습 한 문장
```

## 3. 화면 구성 (섹션별)

### 3-1. 헤더 / 커버

- 단원명·차시·성취기준 표시 (고정, 수정 불가)
- **이름 입력칸** (학생이 자기 이름 입력)
- **핵심 질문** 강조 박스: `"나는 어떤 사람이며, 나는 왜 소중한 사람일까?"`
- 색상: 하늘파랑 계열 그라디언트

---

### 3-2. STEP 1 — 거울 속 나를 만나요 🪞

**목적:** 크롬북 카메라로 자기 얼굴을 관찰하며 도입 흥미 유발

**구현 요소:**

1. **카메라 실행 버튼**
   - `[📷 카메라 켜기]` 버튼 클릭 → `getUserMedia()` API로 웹캠 스트림 활성화
   - 버튼 아래 `<video>` 엘리먼트에 실시간 영상 표시
   - 영상 크기: 최대 360×270 (4:3 비율), 중앙 정렬, border-radius: 12px
   - 좌우 반전(mirror): `transform: scaleX(-1)` 적용

2. **셀카 찍기 버튼**
   - `[📸 사진 찍기]` 버튼 → `<canvas>`로 현재 프레임 캡처
   - 캡처된 이미지를 `<img>`로 화면에 표시 (카메라 영상 위에 대체)
   - `[다시 찍기]` 버튼으로 재촬영 가능
   - 캡처 이미지는 localStorage에 base64로 저장

3. **관찰 안내 말풍선**
   - 교사 발문을 말풍선 UI로 표시:
     - "거울 속 내 표정은 어때 보이나요?"
     - "오늘 기분은 어떤가요?"
   - 말풍선 배경: 연하늘파랑, 아이콘 🤔

4. **브라우저 카메라 미지원/거부 시 대체 화면**
   - 이모지 거울 🪞 애니메이션으로 대체
   - 텍스트: "카메라를 사용할 수 없어요. 손거울을 보며 생각해 보세요!"

---

### 3-3. STEP 2 — 나를 나타내는 낱말 3개 ✏️

**목적:** 자기 자신을 직관적으로 표현하는 낱말 입력 (태그 방식)

**구현 요소:**

1. **태그 입력 영역**
   - 낱말 입력 후 Enter 또는 Space → 태그(chip)로 변환
   - 태그 최대 3개 제한
   - 3개 이미 입력 시 입력창 비활성화 + 안내 메시지: "낱말 3개를 모두 입력했어요! 🎉"
   - 태그 삭제: 태그 우측 ✕ 버튼

2. **태그 스타일**
   - 각 태그: 하늘파랑 그라디언트 pill 형태
   - 태그 추가 시 팝인(scale) 애니메이션

3. **카운터 표시**
   - "낱말 1/3", "낱말 2/3", "낱말 3/3 완성! ✓" 실시간 업데이트
   - 3개 완성 시 카운터 초록색으로 전환

4. **예시 힌트 (접기/펼치기)**
   - 기본 숨김, `[💡 예시 보기]` 버튼으로 토글
   - 예시 칩: "축구 좋아해요", "그림 잘 그려요", "친구를 잘 도와줘요", "수줍음이 많아요"
   - 예시 칩 클릭 → 해당 낱말이 자동으로 태그로 추가

---

### 3-4. STEP 3 — 나는 이런 사람이에요 📝

**목적:** 5개 문장 빈칸 완성 — 자기탐구 핵심 활동

**문장 구성 (고정 prefix + 자유 입력):**

| # | 전반부 (고정) | 입력 placeholder |
|---|-------------|-----------------|
| 1 | 나는 | 을/를 좋아해요. | "예: 축구, 그림 그리기" |
| 2 | 나는 | 할 때 기뻐요. | "예: 친구랑 놀 때, 책 읽을 때" |
| 3 | 나는 | 을/를 잘해요. | "예: 달리기, 수학" |
| 4 | 나는 | 이/가 아직 어려워요. | "예: 발표, 줄넘기" |
| 5 | 나는 친구들이 | 라고 말해 줄 때 기분이 좋아요. | "예: 재미있대요, 친절하대요" |

**구현 요소:**

1. **카드 레이아웃**
   - 문장 1~4: 2열 그리드 (모바일: 1열)
   - 문장 5: 전체 너비 1열
   - 각 카드 배경색: 순서대로 sky-l, mint-l, warm-l, rose-l, gray-50
   - 카드 내부: prefix 텍스트(굵게) + 입력칸(textarea, 1~2줄)

2. **완성 표시**
   - 각 문장 입력 완료 시(글자 1자 이상) 카드 우측 상단에 ✓ 체크 아이콘 표시
   - 5개 모두 완성 시 완료 메시지 배너 표시: "나를 잘 표현했어요! 👏"

3. **특별 강조 발문 박스**
   - 문장 3~4 사이에 삽입:
   - 배경: 따뜻한 노랑, 좌측 선 강조
   - 텍스트: "💬 잘하는 것과 아직 어려운 것이 함께 있어도 나는 소중할까요?"
   - 이 발문은 입력칸 없이 생각만 하는 '멈춤 발문'으로 표시

---

### 3-5. 하단 완성 카드 (STEP 3 이후 자동 출현)

**목적:** 자기탐구 요약 및 정리

**구현:**
- STEP 3의 5개 문장이 모두 채워지면 하단에 슬라이드인 애니메이션으로 등장
- "오늘 내가 발견한 나의 모습은:" 제목
- STEP 3 입력 내용 5개가 자동으로 요약 카드에 미리보기로 표시 (읽기 전용)
- `[다음 활동으로 →]` 버튼: worksheet2.html로 이동 링크

## 4. UX / 인터랙션 규칙

| 항목 | 규칙 |
|------|------|
| 자동 저장 | localStorage, 입력마다 300ms debounce |
| 저장 키 | `ws1_` + 이름 (이름 미입력 시 `ws1_noname`) |
| 내보내기 | 인쇄/PDF 버튼 → `window.print()`, 인쇄용 CSS 별도 |
| 진행률 표시 | 상단 progress bar: STEP1~3 완료율 % |
| 자동 높이 | 모든 textarea: `scrollHeight` 기반 auto-grow |
| 빈칸 제출 방지 | STEP 전환 시 미완료 항목 개수 알림 (강제 차단은 하지 않음) |
| 접근성 | 모든 입력칸에 `aria-label` 부여 |

## 5. 디자인 시스템

### 색상 팔레트

```css
--sky:    #4A90D9   /* 주 색상 — 하늘파랑 */
--sky-d:  #2563A8   /* 진한 파랑 (헤더, 강조) */
--sky-l:  #EAF3FC   /* 연한 파랑 (카드 배경) */
--warm:   #F5A623   /* 강조 오렌지 (버튼, 힌트) */
--warm-l: #FEF3DC   /* 연노랑 (발문 박스) */
--mint:   #3DB88B   /* 완료/성공 색상 */
--mint-l: #E5F7F1   /* 연민트 (카드 배경) */
--rose:   #E05C7A   /* 포인트 로즈 */
--rose-l: #FCEEF2   /* 연로즈 (카드 배경) */
--gray-50:#F8FAFB
--gray-200:#E0E6ED
--gray-600:#5A6878
--gray-800:#2C3A4A
```

### 폰트
- 본문: `Noto Sans KR` (Google Fonts)
- 제목 폰트 사이즈: 22px (커버), 17px (섹션), 14px (카드)
- 한글 line-height: 1.75

### 반응형
- 기준 max-width: 680px (크롬북 화면 고려)
- 모바일 breakpoint: 540px

### 애니메이션
- 섹션 등장: `fadeUp` (translateY 16px → 0, 0.4s)
- 태그 추가: `popIn` (scale .7 → 1, 0.18s)
- 완성 배너: `slideDown` (translateY -20px → 0, 0.35s)
- 카메라 없을 때 거울 이모지: `float` (수직 8px 왕복, 3s)

## 6. 인쇄/PDF 스타일 (`@media print`)

- 상단바, 진행바, 버튼류 숨김
- 카메라 영상/이미지 영역: 캡처된 이미지는 인쇄, 없으면 빈 점선 박스
- 태그칩: 인쇄 시 색상 유지 (`print-color-adjust: exact`)
- 문장 카드: 배경색 유지, 입력 내용 그대로 출력
- A4 세로 기준, 여백: 상하 15mm 좌우 18mm
